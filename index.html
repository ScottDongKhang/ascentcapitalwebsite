What We Built
The idea

Most systems act.
Ours thinks first.

We started with a question: what does a genuinely honest trading system look like? One that doesn’t just fire on its own signals — but actually challenges them, validates them, and knows when to stop. Every design decision in Ascent traces back to that question.

The approach

Built in layers.
Each one earns its place.

Data, signals, portfolio construction, market regime detection, a pre-trade debate, live execution, and a self-learning layer still being built. Each component has a distinct job and clean outputs. We found a flaw in our validation process that made our numbers look better than they were — we fixed it and kept the honest ones. That standard runs through everything we build.

The core innovation

It argues with itself
before it spends money.

Before any trade executes, five AI agents debate it. A judge synthesizes the argument and issues a verdict. The system tracks what it has gotten wrong historically and injects that knowledge into every future debate. It cannot unknow its own mistakes.

The direction

A system that learns
from being wrong.

We’re building a layer that lets the system identify its own structural failures, propose architectural changes, test them against the live platform, and integrate them only if the evidence supports it. Every change is logged permanently. The system can explain why it is the way it is.

Debate Layer

Before every trade,
five agents disagree.

Most trading systems treat the model’s output as a decision. We treat it as a proposal — one that has to survive a structured argument before anything executes.

Each agent has a specific role: make the bull case, argue against it, find the thing the model most likely missed, assess whether the current environment supports the trade, and check the mechanics. A judge synthesizes all of it: proceed, reduce size, or halt. No verdict, no trade.

There’s also a component we’re particularly proud of: a blind spot detector that reads every past verdict and finds patterns the system keeps repeating. Those get injected as permanent context into every future debate. The system can’t look away from what it’s been wrong about.

How It Learns

What if the system
could fix itself?

Most trading systems have a fixed architecture. Someone writes the logic, it runs, they tune it. The assumptions baked into the code never really change — they just get weighted differently.

We’re building something different. Ascent keeps a live record of every assumption it’s making about markets — stated explicitly, so they can be tested. When a failure pattern shows up repeatedly, the system proposes a structural change, tests it in parallel with the live platform, and only integrates it if the evidence supports it. Every decision is logged permanently.

This layer is still in development. We’re being honest about that. But the design is real, and the direction is clear.

The learning cycle
Failure is detected
The blind spot detector identifies a pattern the system keeps repeating across past decisions.
A change is proposed
The system generates a specific structural proposal to address the failure. No guessing — a testable hypothesis.
It runs in the shadow
The proposed change runs in parallel with the live system — seeing the same data, generating its own output, but not affecting any real decisions.
Evidence decides
Only if the shadow results support the change does it get integrated. Every outcome is logged permanently — accepted or rejected.
Principle 01
Say what you believe, so you can be wrong
Every assumption the system makes about market behavior is written out explicitly — not buried in code. If it’s stated, it can be tested. If it keeps failing, it can be replaced.
Principle 02
Changes have to earn their place
No structural change is promoted until it proves itself against the live system in a shadow testing period. The burden of proof is on the new idea, not the existing one.
Principle 03
Every decision gets a permanent record
Every architectural change is logged: what triggered it, what was changed, and what happened. The system can explain why it is the way it is, all the way back to the beginning.

System Architecture

For those who want to go deeper. Each layer runs independently, with defined inputs and outputs. Click any to expand.

L11Hypothesis MemoryBuilding
Every structural change the system makes to its own architecture is stored permanently — what triggered it, what was changed, and what happened afterward. The system can answer why it is the way it is, all the way back to the beginning.
L10Hypothesis EngineBuilding
When a failure pattern crosses a threshold in the blind spot map, this layer generates a proposed architectural change, routes it to shadow testing, and only promotes it if the out-of-sample evidence supports it. The promotion threshold scales with the size of the change.
L9Debate + Blind Spot DetectorLive
Five agents, one judge, three possible verdicts. A separate process continuously scans the full history of past verdicts to identify systematic failure patterns and injects them as permanent context into every future debate session.
L8Multi-Asset OrchestratorLive
Merges outputs from all four specialist desks into a single portfolio decision. Capital allocation across desks adjusts dynamically with the current market regime — not a static split.
L7Specialist Desks (US / Macro / Intl / Alt)Live
Four desks, each with its own signal logic, risk constraints, and asset universe. US equities runs the full research pipeline. Macro covers rate-sensitive assets. International covers developed and emerging markets. Alternatives covers real assets and vol-adjacent instruments.
L5–6Execution + Risk ControlsLive
Runs on a daily automated schedule. Converts target weights into orders. A drawdown-triggered kill switch can halt all execution if risk conditions are breached. Every run is logged in full.
L4Market Regime DetectionLive
Continuously infers the current market environment and translates it into a posture label every downstream layer can act on. Portfolio construction, desk allocation, and debate context all shift based on the current regime.
L3Portfolio ConstructionLive
Converts alpha scores into actual position sizes under diversification and concentration constraints. Every rebalance writes a full ledger — not just summary metrics — so post-hoc inspection is always possible.
L2Signal GenerationLive
Multiple theories about market behavior run simultaneously and combine into a composite signal. Each theory is stated as an explicit, testable hypothesis — so if it keeps being wrong, the debate and hypothesis layers can identify and address it.
L1Data + FeaturesLive
Price and macro data with point-in-time joins to prevent look-ahead contamination. Historical universe logic accounts for the fact that the set of securities that existed at any given date differs from what exists today — a subtle issue most backtests quietly ignore.
